# v1.0.0 — Frixty Fetcher's first public release

A Chrome extension that downloads videos and images from the active
tab. Supports YouTube, Reddit, Twitter / X, Instagram, Facebook, and
TikTok with site-specific extractors; falls back to yt-dlp's generic
extractor for hundreds of other sites.

Free, open source (MIT), no telemetry, no servers. Everything runs
locally.

### Install

1. Pick the installer for your OS from the **Assets** below and run
   it. It writes a Go native messaging host + bundled `yt-dlp` and
   `ffmpeg` to:
   - **Windows:** `%LOCALAPPDATA%\frixty-fetcher\`
   - **macOS:** `~/Library/Application Support/frixty-fetcher/`
   - **Linux:** `~/.local/share/frixty-fetcher/`
2. Open `chrome://extensions` (or `brave://extensions`,
   `edge://extensions`), enable **Developer mode**, click **Load
   unpacked**, and select the `extension/` folder from a checkout of
   this repo (or a download of [the source zip](https://github.com/ilobaak/frixty-fetcher/archive/refs/tags/v1.0.0.zip)).
3. Click the toolbar icon. Open a YouTube / Reddit / Twitter / IG /
   Facebook / TikTok page first.

> **Why are the installers so large?** The Windows installer is
> ~232 MB, macOS ~129 MB, Linux ~93 MB. They embed `yt-dlp` and
> `ffmpeg` so install needs zero pre-existing dependencies — ffmpeg
> alone accounts for most of the size (Windows ffmpeg is ~120 MB
> static-linked). The frixtyhost itself is ~8 MB; everything else
> in the installer is the bundled tooling.

> **Linux notes.** After downloading `installer-linux`, mark it
> executable (`chmod +x installer-linux`) before running. The
> installer auto-detects Chrome, Chromium, Brave, Edge, Vivaldi,
> and Opera — manifests are written into every browser dir that
> exists. The host's "Save As" / folder picker uses `zenity` (with
> automatic fallback to `kdialog` on KDE); on a minimal install
> without a desktop environment, choose Options → "Specific folder"
> save mode to avoid the dialog entirely.

> **First-run warnings.** The installer is unsigned for v1.0.0.
> - Windows shows "Windows protected your PC" → click **More info → Run anyway**.
> - macOS shows "cannot verify the developer" → right-click the
>   installer → **Open** to bypass once.
> Code-signing certificates are on the roadmap.

### What's in this release

- **Site-specific extractors** for YouTube (videos + Shorts + on-page
  grab button), Reddit (videos + single images + galleries), Twitter /
  X (photos + videos with quality variants + per-tweet grab button),
  Instagram (feed posts + carousels + reels + stories + per-post grab
  button), Facebook (videos + reels + photo albums + floating grab
  button), and TikTok (3-tier URL extraction with MAIN-world fetch
  interceptor + on-page progress ring).
- **Lazy-probe gate** — opening the popup on a non-supported site
  (e.g. Vimeo) shows a Fetch prompt instead of auto-spending several
  seconds in yt-dlp on a page where the extractor likely won't apply.
  Click Fetch to opt in. Once a host succeeds, future popup opens
  skip the prompt for the rest of the session.
- **Picker state persists across popup close.** Closing and reopening
  the popup on the same page restores whatever you fetched last —
  no more clicking Fetch twice. Cleared automatically when the active
  tab navigates to a different page or when the picker is dismissed
  via its × button.
- **Optional subtitle download.** A "Include Subtitles File (if
  available)" checkbox in the video picker. Off by default — explicit
  `--no-write-subs` is passed so a user-level yt-dlp config can't
  sneak a `.vtt` past you. When checked, both explicit + auto-
  generated subtitle tracks are written; on a video with no captions,
  yt-dlp skips silently (no error, no crash).
- **Single-extension downloads.** When you pick a filename in Save
  As ending in `.mp4` / `.mkv` / `.webm` / `.mov`, the host strips
  the literal extension and lets yt-dlp pick the best-quality
  streams + actual container. You always get top quality AND a
  single clean extension — no more `Video.mp4.mkv` artifacts.
- **Cookie support per site** — the popup honours your browser's
  logged-in session cookies for downloads that require auth (age-gated
  YouTube, private Twitter accounts you follow, etc.). Per-site
  toggles in Options (Always / Auto / Never).
- **Save-as / specific-folder / default-folder** save modes with
  filename-template controls. Galleries write into a sanitized
  album-folder.
- **yt-dlp self-update** — bundled yt-dlp updates in the background
  every 12 hours; manual "Check for updates" button in Options.
- **Frixty Fetcher self-update** — a "Check for updates" button on
  the helper itself, pointed at this repo's GitHub Releases. The new
  binary takes effect on next host launch.
- **Privacy-first.** No analytics, no telemetry, no third-party
  servers. Cookies leave your browser only via the local pipe to the
  helper (which writes them to a temp file with restricted
  permissions). See [PRIVACY.md](https://github.com/ilobaak/frixty-fetcher/blob/main/PRIVACY.md).
- **Cookies are partition-aware** so Chrome 124+'s third-party-cookie
  partitioning (CHIPS) doesn't silently lock yt-dlp out of
  authenticated requests.

### Built-in checksum verification

The installer's bundled `yt-dlp` and `ffmpeg` binaries are downloaded
from upstream at build time with their published checksums verified
before being embedded. The yt-dlp self-updater verifies against
upstream's `SHA2-256SUMS` release asset on every check.

### Known limitations

- Age-gated YouTube videos require login (set YouTube cookies to
  "Always" or "Auto" in Options).
- Private Twitter / Instagram accounts you don't follow are
  inaccessible — yt-dlp can only see what your logged-in account can.
- Facebook gated content sometimes fails; reload the page and try
  again.
- TikTok logged-out feeds rely on the MAIN-world fetch interceptor;
  scroll past a post once before clicking the grab button to give the
  interceptor time to capture metadata.
- The installer is unsigned (see "First-run warnings" above).

Full list in the [project README](https://github.com/ilobaak/frixty-fetcher/blob/main/README.md).
